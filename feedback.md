# Feedback and Support

All feedback and support requests are handled through GitHub Issues. You do not need to be a developer to submit one -- just a free GitHub account.

## Report a Bug

Something broken or not working as expected?

[Report a bug](https://github.com/NotMilk/BackyardSapTracker-support/issues/new?template=bug_report.yml){: .store-link}

Include what you were doing, what happened, and what you expected to happen. Device model and OS version help with troubleshooting.

## Request a Feature

Have an idea for something the app should do?

[Request a feature](https://github.com/NotMilk/BackyardSapTracker-support/issues/new?template=feature_request.yml){: .store-link}

Describe the feature and how you would use it. Practical use cases help prioritize development.

## General Feedback

Questions, comments, or anything else?

[Submit feedback](https://github.com/NotMilk/BackyardSapTracker-support/issues/new?template=feedback.yml){: .store-link}

## Before Submitting

- Check the [FAQ]({{ '/faq' | relative_url }}) to see if your question is already answered.
- Search [existing issues](https://github.com/NotMilk/BackyardSapTracker-support/issues) to avoid duplicates.
- One topic per issue keeps things organized.

## Browse Existing Issues

[View all issues](https://github.com/NotMilk/BackyardSapTracker-support/issues) to see what has been reported, requested, or discussed.
